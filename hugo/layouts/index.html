What is Gardener

Gardener serves distributed Kubernetes clusters

Gardener hosts a multitude of isolated control planes in dedicated Kubernetes clusters and the coresponding nodes are managed on remote infrastructure accounts under the cluster owners control.

Scalable by design

A single Kubernetes cluster can host an enormous amount of control planes. Should that prove insufficent, Gardener can always scale up and orchestrate more control plane clusters on each supported infrastructure without downtimes.

Get The Kubernetes You Really Wanted

The clusters Gardener provisions are like DIY clusters, except you don’t do it yourself
Configurable

Instead of barely configurable control planes and non-conditional updates not on your schedule, Gardener allows its service consumers to decide whether and what they want to control for more advanced use cases and integration scenarios.

100% Kubernetes compliant

Gardener is Kubernetes native and is not shy to be completely transparent on its compliance, proudly holding the 100% badge with a public evidence for that.

The one you already know

Gardener delivers the same Kubernetes you know from kubernetes.io and certified for. The same binaries, the same tools. You are already trained for it.

Infrastructure Agnostic

Focus on Kubernetes, not on the infrastructure

One Single Cluster Fleet Operations Hub

Gardener is designed as a one-stop shop for all your cluster fleet management operations on multiple infrastructures.

Homogeneous Operations and Interface

No mandatory vendor-specific tools to hook you on. Instead, the same slick dashboard UI, a standard kubectl CLI or a REST API is all it takes to get the job done reagrdless of the infrastructure. Or, if you are Gardener service operator, you can use gardenctl, our CLI tool for cluster operations.

Comparable experience

Operations experience with infrastructures varies. Gardener makes up for their deficencies as much as possible to provide comparable experience on each supported infrastructure.

External DNS and Certificates Management

Gardener takes infrastructure abstraction to a lp level modelng DNS records and TLS certificates for your ingress endpoints as Kubernetes resources, so that you can manage them right frоm your clusters. You can start right away with the most popular DNS service and certificate providers or you can add your own.

Common Host OS for Gardener Managed Cluster Nodes

Instead of dealing with each infrastructure provider OS, choose from Gardener’s out-of-the-box set - SUSE CHOST, Canonical Ubuntu, Flatcar Container Linux. Or employ Garden Linux, optimized specifically for this job. Or just bring your own!
